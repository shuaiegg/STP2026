# Brainstorming Templates

Use these templates to structure the brainstorming session.

## 1. Feature Definition Framework (User Story)

**Format:**
"As a [Type of User], I want to [Action] so that [Benefit]."

**Checklist:**
- [ ] Who is the primary user?
- [ ] What is the core problem being solved?
- [ ] What is the definition of done?
- [ ] enhance with acceptance criteria.

## 2. SWOT Analysis (For Strategic Features)

**Strengths:**
- What internal advantages does this feature leverage?
- Does it fit well with our current tech stack?

**Weaknesses:**
- What complexities does it introduce?
- Are there technical debt risks?

**Opportunities:**
- Does it open new markets or user segments?
- Can it be monetized?

**Threats:**
- Are there better alternatives?
- Does it distract from the core product?

## 3. Technical Feasibility Check

**Backend:**
- [ ] Data model changes required?
- [ ] API endpoints needed (REST vs GraphQL)?
- [ ] Performance implications (e.g., N+1 queries)?

**Frontend:**
- [ ] UI components needed (reuse vs build)?
- [ ] State management requirements?
- [ ] Client-side validation logic?

**Integration:**
- [ ] 3rd party APIs involved?
- [ ] Authentication/Authorization scopes needed?

## 4. "The Amazon Press Release" Approach

*Draft a hypothetical press release for the feature to clarify its value proposition.*

- **Headline**: One sentence description.
- **Customer Quote**: How a user would describe the benefit.
- **The Problem**: What current pain point exists?
- **The Solution**: How does this feature solve it?
