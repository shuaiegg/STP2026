# STP2026: Site Intelligence 深度进化方案 (Internal-Driven Strategy)

> **Version**: 1.1  
> **Status**: Strategic Execution Plan  
> **Author**: Aladdin (Manus-Soul v1.0)  
> **Focus**: 从“数据爬取”向“逻辑推演”跨越

## 1. 核心愿景：业务逻辑的数字孪生 (Digital Twin)
STP2026 不再仅仅是分析站点的 SEO 工具，而是一个能够理解业务“公理系统”并自动诊断“语义债”的智能合伙人。系统将通过 **内生驱动战略 (Internal-Driven Strategy)**，自主发现业务逻辑缺口并驱动内容资产增值。

---

## 2. 模块进化方案

### A. 业务 DNA 萃取引擎 (The Axiom Extractor)
**从“描述”转向“公理建模”**
*   **进化点**：除了提取关键词，更要提取 **业务命题 (Business Propositions)**。
*   **执行逻辑**：
    *   **多维采样**：在 Homepage/About 基础上，强制采样 `Pricing`, `Solutions`, `Case Studies` 页面。
    *   **逻辑链识别**：识别 “Problem -> Solution -> Proof” 结构。
    *   **产出物**：`BusinessOntology.json`（包含业务公理、核心差异化、价值主张）。

### B. 语义债量化模型 (The Semantic Debt Model)
**将“缺口”数字化与度量化**
*   **进化点**：引入 **逻辑闭环率 (Logic Completion Rate)**。
*   **执行逻辑**：
    *   **压力测试**：利用 LLM 模拟理想客户视角，对当前站点内容进行“逻辑质询”。
    *   **熵值分析**：计算内容分布的混乱度，量化 Topical Authority 的强弱。
    *   **关键指标**：
        *   **Coverage Score**: 业务逻辑点的覆盖率。
        *   **Proof Density**: 每一个逻辑点是否有足够的案例/数据支撑。

### C. 3D 战略沙盘与虚影节点 (Ghost Nodes)
**从“现状展示”转向“预测推演”**
*   **进化点**：在 3D 星图中引入 **意图流 (Intent Flow)** 与 **虚影节点 (Ghost Nodes)**。
*   **执行逻辑**：
    *   **常识拓扑匹配**：将提取的业务 DNA 与行业标准知识库比对，预测“该有但缺失”的节点。
    *   **虚影呈现**：在星图中以半透明形式标注逻辑缺口。
    *   **一键闭环**：点击 Ghost Node，自动调用 `StellarWriter` 生成补全该逻辑缺口的内容任务。

## 3. 核心技术路径 (Actionable Milestones)

### Phase 1: 逻辑感知层 (Immediate)
*   **升级 `extractBusinessDna`**：
    *   增加对 `Pricing` 和 `Case Study` 路径的启发式搜索。
    *   提示词优化：从“总结关键词”改为“推导业务成立的核心逻辑链”。
*   **引入 `Logic Auditor` 函数**：在聚类完成后，评估每个聚类对业务 DNA 的支撑权重。

### Phase 2: 预测生成层 (Short-term)
*   **Ghost Node 算法实现**：
    *   输入：`BusinessOntology` + `CurrentClustering`。
    *   输出：`MissingLogicPoints[]`（带坐标信息的虚影节点数据）。
*   **星图交互升级**：在渲染层支持非实体节点（Ghost Nodes）的展示与任务绑定。

### Phase 3: 自动驾驶层 (Mid-term)
*   **自愈闭环**：当 GSC 监测到某个核心逻辑节点的流量下降或排名不稳时，自动将该节点状态标记为 `Logical Refactoring Needed`，并排入优化日程。

---

## 4. 业务价值总结 (Business ROI)
1.  **避开红海竞争**：不盲目跟从竞争对手，而是通过补全自身逻辑拼图实现 **主题权威 (Topical Authority)**。
2.  **资产定向增值**：每一分内容预算都花在“填补逻辑真空”上，确保内容即资产。
3.  **决策自动化**：从“我想写什么”进化为“系统告诉我业务需要写什么”。
