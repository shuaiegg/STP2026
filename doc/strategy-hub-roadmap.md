# 站点智能管家：全渠道智能中心 (SEO & SEM Hub) 演进规划

> **Status**: Draft / Vision & Architecture Planning
> **Core Principle**: Actionable Insights & Human-in-the-Loop Automation

本文档详细规划了“站点智能管家” (Site Intelligence) 未来的战略演进路线，核心是从“单一的 SEO 洞察工具”跃升为“SEO + SEM 统一的战略指挥中心”。

---

## 🚀 阶段一：战略行动枢纽 (Strategy Hub)

核心痛点：系统通过 AI 发现了“高优语义债”，但目前距离“最终产出文章”依然存在过高的摩擦力。用户在全自动 AI 和纯粹空白之间缺乏安全感。

**解决方案：引入“人工干预”的可视化战略计划系统，衔接 洞察 与 GEO Writer。**

### 1.1 交互形态：主题支柱泳道看板 (Pillar-Cluster Board)
放弃纯粹的“思维导图”和传统的“Trello 看板”，采用具备 SEO 战略深度与敏捷执行力的**“平铺结构树”**交互形态。

*   **列头 (Columns) = 主题支柱 (Pillar)**
    *   每一列代表一个核心的战略方向或“语义债”（例如：“B2B 自动化营销策略”）。
    *   列头会醒目标注该话题的优先级（如：🔥高搜索、低点击）以及相关的 GSC 数据。
*   **卡片 (Cards) = 集群文章 (Cluster Articles)**
    *   在每个 Pillar 列下，竖向陈列由 AI 根据该话题发散出的 3-5 篇具体“集群文章”建议。
    *   卡片呈现：拟定引流标题、核心打怪关键词、当前状态（🔵计划中、🟡撰写中、🟢已发布）。
*   **核心交互与业务闭环**
    *   **人工精修**：允许客户拖拽卡片更改优先级、删除不想要的话题、修改 AI 拟定的标题和关键词（增强用户控制感）。
    *   **一键派发**：每张卡片具备【⚡ 生成文章】按钮，点击后携带着整个 Pillar 下的背景知识（Business DNA），无缝推送到 GEO Writer 开始自动撰写。

### 1.2 阶段一全局 UI 规划 (UI Blueprint Phase 1)
*   **入口优化**：在控制台主导航栏 (Sidebar) 提升 `[站点智能管家]` 的层级。
*   **全局数据大盘 (/dashboard)**：移除“能源与调用量”这类操作级流水账，替换为具有全局掌控感的宏观指标卡片，如：**资产总数**、**跨站点的全局高优语义债总数**。
*   **新增子导航 (Tabs)**：进入具体站点后，增设名为 **`[战略计划 (Strategy)]`** 的核心视图承载“泳道看板”。现有的 `[概览 (Overview)]` 弱化为纯体检报告。

## 🌌 阶段二：全渠道营销中心 (Omnichannel Hub)

核心愿景：打破自然流量与付费流量的壁垒，接入 Google Ads / Meta Ads 等平台，实现“哪里的钱白花了”与“哪里的转化最好”的全局透视。

### 2.1 数据大一统与跨平台归因
*   打通多渠道的 API (Google Ads / Meta Ads)。
*   抽象为统一的事件流，以“落地页 (Landing Page)”为核心枢纽，展示不同流量来源对同一页面的转化贡献度。
*   **解耦流量视角**：UI 上重构指标大盘，区分为 `[自然流量 (Organic)]` 和 `[付费流量 (Paid)]`，并提供“跨渠道漏洞雷达图”。

### 2.2 AI 智能顾问的演进能力
1.  **广告与落地页语义校验**：自动读取 Ads 的广告文案，再比对对应的着陆页，一旦发现语义不匹配（跳出率隐患），立刻预警。
2.  **独创流量套利雷达 (Traffic Arbitrage)**：扫描高单价 (CPC) 且表现并不极端的广告词，建议用户使用 GEO Writer 迅速铺设自然长尾内容，实现“用算力替换广告费”的市场降维打击。
3.  **自动跨渠道转移建议**：若某个关键词的 SEO 自然排名极其稳定，AI 将主动建议削减其在该词的保护性 SEM 预算，转移给转化率高但自然排名落后的弱势词组。

### 2.3 AI 接管与自主代播 (Agentic Execution)
*   **自动化测试 (Agentic A/B Testing)**：用户批准内容后，自动生成不同的推广物料 (H1 标题或 CTA)，分发给一定比例流量后，系统自动基于 CTR 保留更优版本。
*   **带护栏的任务调度 (Human-in-the-loop Agent)**：不直接动用客户的资金盘。AI 若发现“某条亏钱的广告”，会生成一张卡片在控制面板提示：“建议暂停此广告系列，预计挽回 $100 止损。是否 [一键执行]？”。

### 2.4 阶段二核心 UI 突破 (UI Blueprint Phase 2)
*   **流量视角解耦**：当 SEM 接入后，数据图表不能混为一谈。站点详情的二级导航将 `[搜索表现]` 演变为两个并列级：**`[自然增长 (Organic)]`** 与 **`[付费增长 (Paid Ads)]`**。
*   **跨通道漏斗图 (Omnichannel Funnel)**：在站点的 `[概览]` 页引入全新的可视化组件。展示有多少漏斗底部的转化来自于长期滋养的自然流量，哪些靠高昂的竞价续命。
*   **常驻的“AI 营销副总裁”组件 (Agent Mode UI)**：
    *   在桌面的右下角或全局顶部状态栏加入一个常驻悬浮 UI（发光提示）。
    *   不是被动的数据图表，而是主动发起对话式的通知组件。只有触发优化阈值（如发现高ROI缺口、发现广告亏损）才会亮起弹出。
    *   界面语言从“数据汇报”变为“拟人化决策”（如：“经分析，为您过去3天省下了 $200 预算，另外规划了 3 篇引流文章，是否一键执行？”）。

## ⚠️ 核心挑战与应对策略 (Risk Mitigation)

1.  **归因争议之战 (Attribution Nightmare)**：针对 Google Ads 和 GA4 的转化数据扯皮，最初期在 UI 上透明化归因模型（如明确展示：基于 GA4 Last-Click 数据），中期再引入系统推演的“辅助转化 (Assisted)”归因建议。
2.  **数据虚假关联判断**：确保大模型的 Prompt 具备“统计显著性 (Statistical Significance)”思维。仅对趋势稳定（如持续 X 天相关）且有内在逻辑的原因报警，防止制造无用焦虑以及盲目让用户烧钱。
3.  **封号责任红线 (Liability)**：通过 API 对大平台（如 Google Ads）进行修改极易触碰风控。所有的代操作在初期必须严格遵循“AI 提案 -> 客户点击确认同意 -> 机器代发”的手动阻断模式，切不可全自动放飞自我。
