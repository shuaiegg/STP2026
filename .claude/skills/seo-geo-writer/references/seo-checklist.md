## Meta Tags

### Title Tag
- [ ] Length: 50-60 characters
- [ ] Primary keyword front-loaded
- [ ] Brand name at end (optional)
- [ ] Unique across site
- [ ] Compelling for CTR

**Template:**
```
{Primary Keyword} - {Secondary Benefit} | {Brand}
```

**Examples:**
```
AI Marketing Automation Guide - Scale Your Growth | Brand
出海营销自动化教程 - 2024完整指南 | Brand
```

### Meta Description
- [ ] Length: 120-160 characters
- [ ] Contains primary keyword naturally
- [ ] Includes clear CTA
- [ ] Unique value proposition
- [ ] No keyword stuffing

**Template:**
```
{What it is}. {Key benefit}. {CTA}.
```

## Content Structure

### Heading Hierarchy
- [ ] Single H1 containing primary keyword
- [ ] H2s for major sections (3-7 recommended)
- [ ] H3s for subsections
- [ ] No skipped levels (H1 → H3)
- [ ] Question-format H2s for FAQ intent

### Paragraph Structure
- [ ] First paragraph: Direct answer (≤50 words)
- [ ] Paragraphs: 2-4 sentences max
- [ ] One idea per paragraph
- [ ] Active voice preferred

### Lists and Tables
- [ ] Bullet/numbered lists for scanability
- [ ] Tables for comparisons/data
- [ ] Alt formats: flowcharts, diagrams

## URL Optimization

### URL Structure
- [ ] Short (3-5 words ideal)
- [ ] Contains primary keyword
- [ ] Uses hyphens, not underscores
- [ ] Lowercase only
- [ ] No special characters
- [ ] No dates (unless news)

**Good:**
```
/ai-marketing-automation-guide
/出海营销自动化教程
```

**Bad:**
```
/post/2024/01/15/ultimate-guide-to-ai-marketing-automation-for-beginners
```

## Image Optimization

### File Names
- [ ] Descriptive, keyword-rich
- [ ] Hyphens between words
- [ ] No generic names (IMG_001.jpg)

**Example:**
```
ai-marketing-workflow-diagram.png
```

### Alt Text
- [ ] Describes image content
- [ ] Contains keyword if natural
- [ ] ≤125 characters
- [ ] No "image of" prefix

### Technical
- [ ] WebP/AVIF format preferred
- [ ] Compressed (<200KB ideal)
- [ ] Lazy loading enabled
- [ ] Explicit width/height

## Internal Linking

### Link Strategy
- [ ] 3-5 internal links per article
- [ ] Link to pillar/cluster content
- [ ] Semantic anchor text (not "click here")
- [ ] Link relevant terms naturally
- [ ] Update old posts to link new content

### Anchor Text Guidelines
- [ ] Descriptive of target page
- [ ] Varied (not exact match every time)
- [ ] Natural within sentence

**Good:**
```markdown
Learn more about [marketing automation workflows](/automation-workflows)
```

**Bad:**
```markdown
[Click here](/automation-workflows) to learn more
```

## E-E-A-T Signals

### Experience
- [ ] First-person case studies
- [ ] "In our experience..." statements
- [ ] Specific numbers and results
- [ ] Screenshots/proof when possible

### Expertise
- [ ] Author byline with credentials
- [ ] Author page with bio
- [ ] Relevant certifications mentioned
- [ ] Technical depth appropriate to topic

### Authoritativeness
- [ ] External citations with links
- [ ] Data sources attributed
- [ ] Industry terminology used correctly
- [ ] Referenced by reputable sources

### Trust
- [ ] Updated date visible
- [ ] Factual accuracy verified
- [ ] Clear distinction: opinion vs fact
- [ ] Contact/about page linked

## GEO Specific

### AI Extractability
- [ ] Direct answer in first 50 words
- [ ] Definition follows "X is..." pattern
- [ ] Key facts in bullet points
- [ ] No ambiguous pronouns in key sentences

### Citation Readiness
- [ ] Unique framework/methodology named
- [ ] Original data or research
- [ ] Clear brand attribution for methods
- [ ] Structured for snippet extraction

### FAQ Optimization
- [ ] Questions match AI query patterns
- [ ] Answers are self-contained
- [ ] Each Q&A independent (no "as mentioned above")
- [ ] Answer length: 40-60 words ideal

## Quick Audit Checklist

Before publishing, verify:

```
[ ] Title tag optimized (50-60 chars)
[ ] Meta description compelling (120-160 chars)
[ ] URL short and keyword-rich
[ ] H1 contains primary keyword
[ ] Direct answer in first paragraph
[ ] 3+ H2 sections
[ ] 3-5 internal links
[ ] All images have alt text
[ ] Author byline present
[ ] Schema markup added
[ ] Mobile-friendly verified
```
